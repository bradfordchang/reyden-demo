Methodology. Before anything runs, a validation pass proves you can actually execute the batch: each warehouse must be visible to you and accept your statements, and every dashboard dataset query is compiled with DESCRIBE QUERY on its dashboard's warehouse — analysis resolves every table and enforces Unity Catalog privileges without reading data, so dashboards or datasets you can't run are skipped with the reason shown. Only then does profiling start: scenario SQL is pulled live from each dashboard's Lakeview datasets (default parameters applied), every statement carries a unique tag to defeat result caches, and each run fires all of a dashboard's queries concurrently — the same burst a real dashboard load sends — through the SQL Statement Execution API on your identity (on-behalf-of authorization). Bars, verdicts, and the headline speedup all compare median dashboard load time — the end-to-end wall-clock of that concurrent burst per warehouse. Per-dataset wins are counted separately (individual query times include queueing, so their ratios can run far above the load ratio on a saturated warehouse). Both lanes warm up before each start gun so cold starts never skew a lap.
